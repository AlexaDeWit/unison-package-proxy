# Registry Proxy Core

Core types and evaluation logic for the registry proxy. All types
are registry-agnostic — the same rules engine, metadata format, and
client interface work with any package registry (npm, PyPI, etc.).

## Types and Abilities

```unison
namespace RegistryProxy

-- ============================================================
-- Package Identity (registry-agnostic)
-- ============================================================

{{ Identifies a package across any registry ecosystem.
A package is either unscoped (a bare name) or scoped
(an organization/scope prefix plus a name). }}
type PackageId
  = PackageId.Unscoped Text
  | PackageId.Scoped Text Text

{{ Render a {type PackageId} as a human-readable string.
Scoped packages are displayed as ''@scope/name''. }}
PackageId.toText : PackageId -> Text
PackageId.toText = cases
  PackageId.Unscoped name -> name
  PackageId.Scoped scope name -> "@" ++ scope ++ "/" ++ name

{{ Check if two {type PackageId} values are equal by comparing
their text representations. }}
PackageId.eq : PackageId -> PackageId -> Boolean
PackageId.eq a b = PackageId.toText a == PackageId.toText b

{{ Extract the scope from a {type PackageId}, if scoped.
Returns ''None'' for unscoped packages. }}
PackageId.getScope : PackageId -> Optional Text
PackageId.getScope = cases
  PackageId.Scoped s _ -> Some s
  PackageId.Unscoped _ -> None

-- ============================================================
-- Internal Metadata (what rules evaluate against)
-- ============================================================

{{ Summary metadata about a package, abstracted from any specific
registry format. This is what the rules engine evaluates against
for package-level decisions. }}
type PackageInfo =
  { packageId : PackageId
  , description : Optional Text
  , license : Optional Text
  , maintainers : [Text]
  , keywords : [Text]
  , homepage : Optional Text
  , repository : Optional Text
  , latestVersion : Optional Text
  }

{{ Metadata about a specific version of a package. Used for
version-level rule evaluation (e.g., checking for install scripts
or known vulnerabilities). }}
type VersionInfo =
  { versionPackageId : PackageId
  , version : Text
  , publishedAt : Optional Text
  , deprecated : Optional Text
  , hasInstallScript : Boolean
  , dependencyCount : Nat
  , integrityHash : Optional Text
  }

{{ The context provided to the rules engine for evaluation.
Contains package-level metadata and optionally version-level
metadata when the request targets a specific version. }}
type RuleContext =
  { contextPackageInfo : PackageInfo
  , contextVersionInfo : Optional VersionInfo
  }

-- ============================================================
-- CVE Data Structures
-- ============================================================

{{ Severity classification for a CVE record. }}
type Severity
  = Severity.Low
  | Severity.Medium
  | Severity.High
  | Severity.Critical

{{ Render a {type Severity} as a human-readable text label. }}
Severity.toText : Severity -> Text
Severity.toText = cases
  Severity.Low -> "low"
  Severity.Medium -> "medium"
  Severity.High -> "high"
  Severity.Critical -> "critical"

{{ A CVE record describing a known vulnerability that affects
one or more versions of a package. Version ranges are represented
as text strings (semver range parsing is deferred). }}
type CVERecord =
  { cveId : Text
  , summary : Text
  , severity : Severity
  , affectedVersions : [Text]
  , fixedVersions : [Text]
  , cvePublishedAt : Optional Text
  }

{{ Ability for looking up CVE records against a vulnerability
database. Handlers may query in-memory caches, external APIs
(e.g., OSV.dev), or local databases. }}
ability CVELookup where
  lookupCVEs : PackageId -> [CVERecord]
  hasActiveCVE : PackageId -> Text -> Boolean

-- ============================================================
-- Rules Engine
-- ============================================================

{{ The result of evaluating a single rule against a {type RuleContext}.

* ''Allow'' with a reason means the package is permitted.
* ''Deny'' with a reason means the package is rejected.
* ''NoDecision'' means this rule does not apply and evaluation
  should continue to the next rule. }}
type RuleDecision
  = RuleDecision.Allow Text
  | RuleDecision.Deny Text
  | RuleDecision.NoDecision

{{ A pure rule: evaluated against package metadata with no
side effects. Pure rules run first and are fast and deterministic.

Built-in pure rules:
* ''AllowAll'' — permits any package (escape hatch)
* ''AllowPackage'' — permits a specific {type PackageId}
* ''AllowScope'' — permits all packages in a given scope
* ''DenyDeprecated'' — rejects deprecated versions
* ''DenyHasInstallScript'' — rejects packages with install scripts }}
type PureRule
  = PureRule.AllowAll
  | PureRule.AllowPackage PackageId
  | PureRule.AllowScope Text
  | PureRule.DenyDeprecated
  | PureRule.DenyHasInstallScript

{{ An effectful rule: may perform IO such as CVE lookups or
external policy checks. Effectful rules only run if no pure
rule has already made a decision.

Built-in effectful rules:
* ''DenyIfCVE'' — rejects packages with known active CVEs
* ''AllowIfNoCVE'' — permits packages with no known CVEs }}
type EffectfulRule
  = EffectfulRule.DenyIfCVE
  | EffectfulRule.AllowIfNoCVE

{{ A complete rule set combining pure and effectful rules.
Pure rules are evaluated first; effectful rules only run
if all pure rules return ''NoDecision''. }}
type RuleSet =
  { pureRules : [PureRule]
  , effectfulRules : [EffectfulRule]
  }

-- ============================================================
-- Registry Client Interface
-- ============================================================

{{ The raw response from a registry operation, including status
code, headers, and body bytes. This is the common response shape
for any HTTP-based registry. The proxy uses ''statusCode'' to
decide whether to serve the response or begin fallback. }}
type RegistryResponse =
  { statusCode : Nat
  , responseHeaders : [(Text, Text)]
  , responseBody : Bytes
  , contentType : Text
  }

{{ Check whether a status code indicates success (2xx). }}
isSuccess : Nat -> Boolean
isSuccess code =
  (code >= 200) && (code < 300)

{{ A registry client interface. Different registries (npm, PyPI, etc.)
provide different implementations of this record.

The proxy lifecycle using a {type RegistryClient}:

1. Call ''fetchMetadata'' or ''fetchArtifact'' on an upstream client.
2. Check ''statusCode'' of the {type RegistryResponse}.
3. If 2xx, forward ''responseBody'' to the caller.
4. If non-2xx, begin fallback with a different client.
5. On fallback success, use ''parsePackageInfo'' to extract metadata
   for the rules engine.
6. If the rules engine allows, forward the fallback response body.

The ''parse*'' functions are pure — they extract internal metadata
from a {type RegistryResponse} without performing IO. Each registry
implementation provides its own parsing logic (e.g., npm JSON codecs,
PyPI HTML index parsing). }}
type RegistryClient =
  { fetchMetadata :
      PackageId -> '{IO, Exception} RegistryResponse
  , fetchArtifact :
      PackageId -> Text -> '{IO, Exception} RegistryResponse
  , publishArtifact :
      PackageId -> Text -> Bytes -> '{IO, Exception} (Either Text ())
  , parsePackageInfo :
      RegistryResponse -> Either Text PackageInfo
  , parseVersionInfo :
      RegistryResponse -> Text -> Either Text VersionInfo
  , parseVersionList :
      RegistryResponse -> Either Text [Text]
  }
```

## Rule Evaluation

```unison
namespace RegistryProxy

-- ============================================================
-- Pure Rule Evaluation
-- ============================================================

{{ Evaluate a single {type PureRule} against a {type RuleContext}.
Returns ''Allow'', ''Deny'', or ''NoDecision''. }}
evaluatePureRule : PureRule -> RuleContext -> RuleDecision
evaluatePureRule rule ctx =
  pkgInfo = RuleContext.contextPackageInfo ctx
  pkgId = PackageInfo.packageId pkgInfo
  match rule with
    PureRule.AllowAll ->
      RuleDecision.Allow "allowed by AllowAll rule"
    PureRule.AllowPackage targetId ->
      if PackageId.eq pkgId targetId then
        RuleDecision.Allow ("allowed: package " ++ PackageId.toText pkgId)
      else RuleDecision.NoDecision
    PureRule.AllowScope targetScope ->
      match PackageId.getScope pkgId with
        Some s ->
          if s == targetScope then
            RuleDecision.Allow ("allowed: scope @" ++ s)
          else RuleDecision.NoDecision
        None -> RuleDecision.NoDecision
    PureRule.DenyDeprecated ->
      match RuleContext.contextVersionInfo ctx with
        Some vi ->
          match VersionInfo.deprecated vi with
            Some reason ->
              RuleDecision.Deny ("denied: deprecated (" ++ reason ++ ")")
            None -> RuleDecision.NoDecision
        None -> RuleDecision.NoDecision
    PureRule.DenyHasInstallScript ->
      match RuleContext.contextVersionInfo ctx with
        Some vi ->
          if VersionInfo.hasInstallScript vi then
            RuleDecision.Deny "denied: package has install script"
          else RuleDecision.NoDecision
        None -> RuleDecision.NoDecision

{{ Evaluate a list of pure rules against a {type RuleContext},
short-circuiting on the first decisive result. Returns ''None''
if all rules return ''NoDecision''. }}
evaluatePureRules : [PureRule] -> RuleContext -> Optional RuleDecision
evaluatePureRules rules ctx = match rules with
  [] -> None
  rule +: rest ->
    match evaluatePureRule rule ctx with
      RuleDecision.NoDecision -> evaluatePureRules rest ctx
      decision -> Some decision

-- ============================================================
-- Effectful Rule Evaluation
-- ============================================================

{{ Evaluate a single {type EffectfulRule} against a {type RuleContext}.
May perform {type CVELookup} operations. }}
evaluateEffectfulRule : EffectfulRule -> RuleContext -> {CVELookup} RuleDecision
evaluateEffectfulRule rule ctx =
  pkgId = PackageInfo.packageId (RuleContext.contextPackageInfo ctx)
  match rule with
    EffectfulRule.DenyIfCVE ->
      match RuleContext.contextVersionInfo ctx with
        Some vi ->
          if hasActiveCVE pkgId (VersionInfo.version vi) then
            RuleDecision.Deny
              ("denied: active CVE for " ++ PackageId.toText pkgId
                ++ "@" ++ VersionInfo.version vi)
          else RuleDecision.NoDecision
        None ->
          match lookupCVEs pkgId with
            [] -> RuleDecision.NoDecision
            _ ->
              RuleDecision.Deny
                ("denied: active CVEs found for "
                  ++ PackageId.toText pkgId)
    EffectfulRule.AllowIfNoCVE ->
      match RuleContext.contextVersionInfo ctx with
        Some vi ->
          if hasActiveCVE pkgId (VersionInfo.version vi) then
            RuleDecision.NoDecision
          else
            RuleDecision.Allow
              ("allowed: no CVEs for " ++ PackageId.toText pkgId
                ++ "@" ++ VersionInfo.version vi)
        None ->
          match lookupCVEs pkgId with
            [] ->
              RuleDecision.Allow
                ("allowed: no CVEs for " ++ PackageId.toText pkgId)
            _ -> RuleDecision.NoDecision

{{ Evaluate a list of effectful rules against a {type RuleContext},
short-circuiting on the first decisive result. Returns ''None''
if all rules return ''NoDecision''. }}
evaluateEffectfulRules :
  [EffectfulRule] -> RuleContext -> {CVELookup} Optional RuleDecision
evaluateEffectfulRules rules ctx = match rules with
  [] -> None
  rule +: rest ->
    match evaluateEffectfulRule rule ctx with
      RuleDecision.NoDecision -> evaluateEffectfulRules rest ctx
      decision -> Some decision

-- ============================================================
-- Full Rule Set Evaluation
-- ============================================================

{{ Evaluate a complete {type RuleSet} against a {type RuleContext}.

Evaluation proceeds in two phases:
1. All pure rules are checked in order (fast, no IO).
2. If no pure rule decided, effectful rules are checked.
3. If nothing decides, the request is denied (deny-by-default). }}
evaluateRuleSet : RuleSet -> RuleContext -> {CVELookup} RuleDecision
evaluateRuleSet ruleSet ctx =
  match evaluatePureRules (RuleSet.pureRules ruleSet) ctx with
    Some decision -> decision
    None ->
      match evaluateEffectfulRules (RuleSet.effectfulRules ruleSet) ctx with
        Some decision -> decision
        None ->
          RuleDecision.Deny
            ("denied by default: no rule allows "
              ++ PackageId.toText
                (PackageInfo.packageId
                  (RuleContext.contextPackageInfo ctx)))
```
