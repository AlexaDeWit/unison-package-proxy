## Auth / Token Provider

```unison
namespace RegistryProxy

{{ How to retrieve an auth token for a registry endpoint.

* ''NoAuth'' — no authentication required (public registries)
* ''Static'' — a hardcoded token (useful for testing or CI)
* ''FromEnv'' — read from an environment variable at request time
* ''Custom'' — arbitrary IO computation to fetch a token dynamically
}}
type TokenProvider
  = TokenProvider.NoAuth
  | TokenProvider.Static Text
  | TokenProvider.FromEnv Text
  | TokenProvider.Custom ('{IO, Exception} (Optional Text))

{{ Resolve a {type TokenProvider} to an optional token string.
Returns ''None'' for {TokenProvider.NoAuth} or when the environment
variable is not set. }}
resolveToken : TokenProvider -> '{IO, Exception} (Optional Text)
resolveToken = cases
  TokenProvider.NoAuth -> do None
  TokenProvider.Static token -> do Some token
  TokenProvider.FromEnv varName -> do
    match IO.getEnv.impl varName with
      Right val -> Some val
      Left _ -> None
  TokenProvider.Custom thunk -> thunk
```

## Rule Set Utilities

```unison
namespace RegistryProxy

{{ An empty {type RuleSet} with no rules. Useful as a default
for ''additionalRules'' when no per-upstream overrides are needed. }}
emptyRuleSet : RuleSet
emptyRuleSet = RuleSet [] []

{{ Merge two {type RuleSet}s by concatenating their pure and
effectful rule lists. Used to combine global rules with
per-upstream additional rules. }}
mergeRuleSets : RuleSet -> RuleSet -> RuleSet
mergeRuleSets a b =
  use List ++
  RuleSet
    (RuleSet.pureRules a ++ RuleSet.pureRules b)
    (RuleSet.effectfulRules a ++ RuleSet.effectfulRules b)
```

## Server Configuration Types

```unison
namespace RegistryProxy

{{ Configuration for a single upstream registry, including the
client interface and any additional rules specific to this
upstream (beyond the global rules). }}
type Server.UpstreamConfig
  = { client : RegistryClient
    , additionalRules : RuleSet
    }

{{ Main proxy server configuration.

* ''privateUpstream'' — checked first; if it returns a successful
  response, served immediately without rules evaluation.
* ''publicUpstream'' — fallback when private misses; responses are
  subject to rules evaluation before serving.
* ''mirrorTarget'' — where allowed public packages are mirrored.
* ''globalRules'' — the deny-by-default rule set applied to all
  public upstream requests.
* ''listenPort'' — TCP port the proxy server listens on.

Per-upstream ''additionalRules'' are merged with ''globalRules''
via {mergeRuleSets} before evaluation. }}
type Server.ProxyConfig
  = { privateUpstream : Server.UpstreamConfig
    , publicUpstream : Server.UpstreamConfig
    , mirrorTarget : RegistryClient
    , globalRules : RuleSet
    , listenPort : Nat
    }

{{ Build the effective rule set for a given upstream by merging
the global rules with the upstream's additional rules. }}
Server.effectiveRules : RuleSet -> Server.UpstreamConfig -> RuleSet
Server.effectiveRules globalRules upstream =
  mergeRuleSets globalRules (Server.UpstreamConfig.additionalRules upstream)

{{ Create a 404 Not Found response with the denial reason as the
body. Used when the rules engine denies a package — from the
client's perspective, the package simply does not exist. }}
Server.denyResponse : Text -> HttpResponse
Server.denyResponse reason =
  HttpResponse.notFound
    |> HttpResponse.setBody (Body.fromText reason)
```
